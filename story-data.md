## ストーリーテンプレート設計

### 20%固定 / 80%自由の設計思想

各ページには「役割」がある（導入・展開・クライマックス等）。この役割だけが固定（20%）で、それ以外の舞台・キャラクターの見た目・出来事の詳細は全て子どもの発想で置き換え可能（80%自由）。

- **固定（20%）**: ページの物語上の役割・骨格。これがないと物語として成立しない最小要素。
- **自由（80%）**: 子どもが自由に変えられるもの。舞台設定、キャラの見た目、道具、出来事の詳細、雰囲気など。

Freeform全文再生成方式を採用し、スロット単位（`prop`/`background`/`event`等）の型分類は使用しない。子どものキーワードを受け取り、ページの役割を守りつつ全文を自由に再生成する。

## キャラクターエージェント設計

改変が発生した際、該当ページに登場するキャラクターのエージェントがそれぞれ反応を生成し、オーケストレーターが**整合性チェック**を並列実行して物語全体の整合性を保ちながら最終テキスト・画像を決定する。

### アーキテクチャ

```
[ライブ改変リクエスト（現在のページ）]
    │
    ▼
[改変エンジン (engine.ts) - modifyCurrentPage]
    │
    ├── 現在のページのテキストを全文再生成（キーワード組み込み）
    │
    ▼
[オーケストレーター (orchestrator.ts)]
    │
    ├── 以下を並列実行（Promise.all でレイテンシ最小化）:
    │
    │   ├── [キャラクター反応取得]
    │   │   ├── 該当ページの登場キャラクターを特定
    │   │   ├── 各エージェントに並列問い合わせ (Gemini 2.5 Flash)
    │   │   │   ├── 桃太郎エージェント → "ドラゴンのたまご？よし、大切に持っていくぞ！"
    │   │   │   ├── 犬エージェント → "わん！あたらしい たからもの だ！"
    │   │   │   └── 猿エージェント → "きをつけろ、なにが うまれるか わからないぞ"
    │   │
    │   └── [整合性チェック (checkConsistency)]
    │       ├── fixedElements が守られているか
    │       ├── previousPages との物語的矛盾がないか
    │       ├── キャラクターの行動に整合性があるか
    │       ├── 問題なし → "OK" → 元テキストをそのまま使用
    │       └── 問題あり → 修正版テキストを返す
    │
    ├── キャラ反応を統合 → imagePromptAdditions 生成
    ├── 整合性チェック結果で最終テキストを決定
    │
    └── 最終出力
        ├── modifiedText: 最終テキスト（修正版 or 元テキスト）
        ├── approved: 整合性チェック合格か
        ├── characterReactions: 各キャラの反応
        ├── imagePromptAdditions: 画像プロンプトへの追加指示
        └── キャラの感情状態（UIの表情変化などに使用可能）

[波及再生成（次ページ進入時）]
    │
    ├── needsContextRegeneration=true のページに進入
    ├── regeneratePageInContext() で文脈に基づき自動再生成
    │   - キーワードなし、前ページまでの改変文脈のみで再生成
    │   - buildContextRegenerationPrompt 使用
    └── 再生成されたテキストを文字送り表示
```

### 性格安定化手法

各キャラクターエージェントには以下を定義し、一貫した反応を生成する:
- **性格（personality）**: キャラの根本的な性質
- **行動パターン（behaviorPatterns）**: 典型的な行動の傾向
- **反応スタイル（reactionStyle）**: 発言のトーンや言い回し
- **関係性（relationships）**: 他キャラとの関係がリアクションに影響

## オーケストレーターのプロンプト設計

```typescript
// orchestrator.ts の核心
// キャラクター反応と整合性チェックを並列実行
async function orchestrate(params: {
  bookId: string;
  bookTitle: string;
  keyword: string;
  targetPage: StoryPage;
  modifiedText: string;
  previousPages: { pageNumber: number; currentText: string }[];
  apiKey: string;
}): Promise<OrchestratorResult> {
  const [reactions, consistencyResult] = await Promise.all([
    // 1. 該当ページのキャラクターに並列問い合わせ
    Promise.all(activeCharacters.map(char =>
      getCharacterReaction(char, keyword, modifiedText, bookTitle, apiKey)
    )),
    // 2. 整合性チェック（fixedElements + previousPagesとの矛盾検証）
    checkConsistency({ bookTitle, pageRole, fixedElements, previousPages, modifiedText, apiKey }),
  ]);

  const finalText = consistencyResult.approved ? modifiedText : consistencyResult.correctedText;
  return { approved: consistencyResult.approved, modifiedText: finalText, characterReactions, imagePromptAdditions };
}

// 整合性チェック（buildConsistencyCheckPrompt使用）
async function checkConsistency(params): Promise<{ approved: boolean; correctedText: string }> {
  const result = await genai.models.generateContent({ model: TEXT_MODEL, ... });
  // result === 'OK' → approved: true
  // それ以外 → approved: false, correctedText: result（修正版テキスト）
}
```

### 整合性チェックプロンプト（buildConsistencyCheckPrompt）

```
あなたは子供向け絵本「${bookTitle}」の物語の整合性を確認する編集者です。

## このページの固定要素（必ず守るべき）
${fixedElements}

## このページの役割
${pageRole}

## これまでの物語
${previousPages}

## 改変後テキスト
${modifiedText}

## 確認項目
1. 固定要素が守られているか
2. これまでの物語と矛盾していないか
3. キャラクターの行動に整合性があるか

整合性に問題がない場合: 「OK」のみ出力
問題がある場合: 修正版テキストのみ出力（説明不要）
```

### 波及再生成プロンプト（buildContextRegenerationPrompt）

キーワードなし版。前ページまでの改変文脈を踏まえてページテキストを自然に再生成する:
- fixedElements / pageRole は守る
- これまでの物語の流れ（改変含む）を踏まえて自然に書き換える
- ひらがな中心、3〜6歳向け、4〜6行程度

### レイテンシ対策

- キャラクターエージェントへの問い合わせは **Promise.all で並列実行**（直列にしない）
- `gemini-2.5-flash-preview-05-20` を使用（0.5-1.0秒で応答）
- 水彩ディゾルブアニメーション（2-3秒）の間に処理を完了させる
- 最悪の場合、キャラ反応なしで改変だけ先に適用し、反応は後追いで表示
